Lenguajes de Marcas

Guía completa: XML, HTML, DTD, XSD, XPath, XQuery, XSLT, RSS y más

¿Qué es un lenguaje de marcas?

Un lenguaje de marcas (markup language) es un sistema para anotar un documento de manera que la notación sea distinguible del texto. Las marcas (etiquetas) describen la estructura y el significado del contenido, pero no su presentación (aunque pueden influirla).

Se diferencian de los lenguajes de programación en que no ejecutan instrucciones lógicas, sino que organizan y describen datos.

Ejemplos conocidos: HTML (para web), XML (para intercambio de datos), RSS (sindicación), SVG (gráficos vectoriales), MathML (fórmulas matemáticas).

Principales lenguajes de marcas

SGML (Standard Generalized Markup Language)

Estándar internacional (ISO 8879) para definir lenguajes de marcas. Es el "padre" de HTML y XML. Define la sintaxis y las reglas para crear lenguajes de marcas personalizados.

Características: muy potente, pero complejo. Hoy en día se usa principalmente a través de sus descendientes.

HTML (HyperText Markup Language)

Lenguaje para crear páginas web. Se basa en SGML (hasta HTML4) o en XML (XHTML). HTML5 es la versión actual, con etiquetas semánticas y soporte multimedia.

Propósito: mostrar información en navegadores, con enlaces (hipertexto).

XML (eXtensible Markup Language)

Subconjunto simplificado de SGML. Diseñado para almacenar y transportar datos. No tiene etiquetas predefinidas; tú defines las que necesites.

Reglas: debe estar bien formado (elemento raíz, etiquetas cerradas, atributos entre comillas) y puede ser validado con DTD o XSD.

XHTML (eXtensible HyperText Markup Language)

HTML reformulado como XML. Es decir, sigue las reglas sintácticas de XML (etiquetas siempre cerradas, atributos con comillas, etc.).

Ventaja: más estricto, evita errores y facilita el procesamiento automático.

Estructura de un documento XML

Bien formado

Un documento XML está bien formado si cumple las reglas sintácticas básicas:

  • Declaración XML opcional pero recomendada: <?xml version="1.0" encoding="UTF-8"?>
  • Un único elemento raíz que contiene todo.
  • Todas las etiquetas deben cerrarse correctamente (o ser autocerradas: <elem/>).
  • Los atributos deben ir entre comillas (simples o dobles).
  • Los nombres de elementos no pueden empezar con número ni contener espacios.
  • Los caracteres especiales (<, >, &, ", ') deben escaparse con entidades (<, >, &, ", ').

Válido

Un documento es válido si, además de estar bien formado, cumple con las reglas definidas en un esquema (DTD o XSD).

<?xml version="1.0"?> <biblioteca> <libro id="1"> <titulo>El Quijote</titulo> <autor>Cervantes</autor> </libro> </biblioteca>

Validación: DTD y XSD

DTD (Document Type Definition)

Sistema clásico de validación. Define qué elementos y atributos pueden aparecer y en qué orden.

<!DOCTYPE biblioteca [ <!ELEMENT biblioteca (libro+)> <!ELEMENT libro (titulo, autor)> <!ELEMENT titulo (#PCDATA)> <!ELEMENT autor (#PCDATA)> <!ATTLIST libro id ID #REQUIRED> ]>

Operadores: * (0 o más), + (1 o más), ? (0 o 1), | (alternancia).

XSD (XML Schema Definition)

Más moderno y potente. Soporta tipos de datos (string, integer, date...), restricciones, herencia, etc.

<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> <xs:element name="biblioteca"> <xs:complexType> <xs:sequence> <xs:element name="libro" maxOccurs="unbounded"> <xs:complexType> <xs:sequence> <xs:element name="titulo" type="xs:string"/> <xs:element name="autor" type="xs:string"/> </xs:sequence> <xs:attribute name="id" type="xs:ID" use="required"/> </xs:complexType> </xs:element> </xs:sequence> </xs:complexType> </xs:element> </xs:schema>

Tecnologías relacionadas

Lenguaje para navegar y seleccionar nodos en un documento XML. Se usa en XSLT y XQuery.

/biblioteca/libro[1]/titulo // primer libro //libro[@id='1'] // libro con id=1 //libro[año > 2000]/autor // autores de libros posteriores a 2000

Lenguaje de consulta funcional para XML, similar a SQL. Estructura FLWOR (For, Let, Where, Order by, Return).

for $libro in //libro where $libro/precio > 20 order by $libro/titulo return <libro>{$libro/titulo}</libro>

Transforma documentos XML a otros formatos (HTML, texto, XML diferente). Usa plantillas.

<xsl:template match="/"> <html> <body> <xsl:for-each select="//libro"> <p><xsl:value-of select="titulo"/></p> </xsl:for-each> </body> </html> </xsl:template>

Sindicación: RSS y Atom

RSS (Really Simple Syndication) y Atom son formatos XML para distribuir contenido actualizado (noticias, blogs, podcasts). Permiten a los usuarios suscribirse y recibir novedades sin visitar cada sitio.

<rss version="2.0"> <channel> <title>Mi Blog</title> <link>https://miblog.com</link> <description>Noticias</description> <item> <title>Artículo 1</title> <link>...</link> <pubDate>Mon, 17 Jan 2026 10:00:00 GMT</pubDate> </item> </channel> </rss>

Ir a modulos guiados

Iniciacion completa de HTML

Secuencia base para aprender HTML en orden y con criterio tecnico.

1) Sintaxis minima

Todo documento necesita etiquetas bien anidadas y estructura valida.

<tag>contenido</tag>

2) Estructura base

Domina head y body antes de crecer la pagina.

<!DOCTYPE html> <html lang="es">...</html>

3) Jerarquia de texto

Usa encabezados y parrafos en orden logico para lectura clara.

<h1>Titulo</h1> <p>Introduccion</p>

4) Navegacion y enlaces

Construye hipertexto con enlaces internos y externos descriptivos.

<a href="#html-modulos">Ver modulos</a>

5) Estructura semantica

Organiza por bloques funcionales para escalar mejor el sitio.

<header></header> <main></main>

6) Datos y formularios

Representa tablas y captura datos sin mezclar responsabilidades.

<table>...</table> <form>...</form>

Módulos guiados (principiante -> intermedio)

Cada módulo incluye una idea clave y un ejemplo práctico para aplicar de inmediato.

Aprendes la base de cualquier pagina: doctype, idioma, codificacion y viewport.

<!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1"> <title>Mi pagina</title> </head> </html>

Construyes contenido legible con encabezados, listas y enlaces con proposito.

<h2>Temario</h2> <ul> <li>Fundamentos HTML</li> <li>Semantica</li> </ul> <a href="#htmlLabTitle">Ir al laboratorio</a>

Usas etiquetas semanticas para separar secciones y mejorar mantenibilidad.

<header>Cabecera</header> <nav>Menu principal</nav> <main> <section>Modulo 1</section> </main> <footer>Pie</footer>

Representas datos reales y formularios accesibles con etiquetas y atributos correctos.

<img src="topologia.png" alt="Topologia de red en estrella"> <table> <tr><th>Servicio</th><th>Puerto</th></tr> <tr><td>HTTPS</td><td>443</td></tr> </table> <label for="correo">Correo</label> <input id="correo" type="email" required>
  • Jerarquia de encabezados sin saltos incoherentes.
  • Uso correcto de semantica y atributos globales.
  • Accesibilidad basica: alt, label y orden de lectura.
  • Preparado para CSS y JavaScript sin estilos inline innecesarios.
Revision rapida: - Estructura valida - Semantica clara - Enlaces y formularios funcionales - Integracion limpia con CSS/JS

Laboratorio inicial de HTML

Practica guiada en tres pasos para consolidar la base.

Paso A: Estructura

<header>Curso HTML</header> <main>...</main> <footer>Contacto</footer>

Paso B: Datos

<table> <tr><th>Modulo</th><th>Estado</th></tr> </table>

Paso C: Formulario

<form> <label for="nombre">Nombre</label> <input id="nombre" required> </form>

Resultado esperado: una pagina HTML5 semantica, valida y lista para aplicar estilos en CSS.

Glosario de términos

Definiciones profesionales de los conceptos clave en Lenguajes de Marcas.

XML

Lenguaje

eXtensible Markup Language. Lenguaje de marcas para almacenar y transportar datos. Permite definir etiquetas personalizadas.

HTML

Lenguaje

HyperText Markup Language. Lenguaje para crear páginas web. Define la estructura semántica del contenido.

DTD

Validación

Document Type Definition. Define la estructura permitida en un documento XML.

XSD

Validación

XML Schema Definition. Esquema moderno para validar XML, con soporte de tipos de datos.

XPath

Consulta

Lenguaje para seleccionar nodos en un documento XML mediante rutas.

XQuery

Consulta

Lenguaje funcional para consultar y transformar XML, con estructura FLWOR.

XSLT

Transformación

Lenguaje para transformar XML a otros formatos (HTML, texto, etc.).

RSS

Sindicación

Really Simple Syndication. Formato XML para distribuir contenido actualizado.

Bien formado

XML

Documento que cumple las reglas sintácticas de XML (elemento raíz, etiquetas cerradas, etc.).

Válido

XML

Documento bien formado que además cumple un esquema (DTD o XSD).

Namespace

XML

Mecanismo para evitar conflictos de nombres mediante prefijos y URIs.

PCDATA

DTD

Parsed Character Data. Contenido de texto que será analizado por el parser.
