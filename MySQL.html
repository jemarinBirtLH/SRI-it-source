Gestión de usuarios, permisos, vistas y buenas prácticas.

Este documento recopila los comandos, ejemplos y buenas prácticas utilizadas durante la administración de usuarios, privilegios y vistas en MySQL.

👥 1. Creación de usuarios

Usuario administrador (solo servidor)
CREATE USER 'jemarin_adm'@'localhost' IDENTIFIED BY 'contraseña_segura_adm'; GRANT ALL PRIVILEGES ON *.* TO 'jemarin_adm'@'localhost' WITH GRANT OPTION; FLUSH PRIVILEGES;

Características: Acceso total desde la propia máquina del servidor. Ideal para tareas de administración.

Usuario normal (solo desde cliente)
CREATE USER 'jemarin_usu'@'192.168.10.20' IDENTIFIED BY 'contraseña_usu';
Verificación: SELECT user, host FROM mysql.user WHERE user = 'jemarin_usu';

👁️ 2. Vistas sobre la base de datos Jardineria

Crear vista combinando Pedidos y Clientes
USE Jardineria; CREATE VIEW vista_pedidos_clientes AS SELECT p.CodigoPedido, p.FechaPedido, p.Estado, c.NombreCliente, c.Telefono FROM Pedidos p JOIN Clientes c ON p.CodigoCliente = c.CodigoCliente;

Esta vista muestra información útil sin exponer datos internos de ambas tablas por separado.

Otorgar SELECT solo sobre la vista
GRANT SELECT ON Jardineria.vista_pedidos_clientes TO 'jemarin_usu'@'192.168.10.20'; FLUSH PRIVILEGES;
Comprobación desde el cliente:
SELECT * FROM vista_pedidos_clientes LIMIT 5; → ✅ funciona.
SELECT * FROM Clientes; → ❌ error 1142 (comando denegado).
Consultar definición de vistas
SELECT TABLE_NAME, VIEW_DEFINITION FROM information_schema.VIEWS WHERE TABLE_SCHEMA = 'Jardineria';

Esta consulta muestra el código SQL con el que fue creada cada vista, útil para auditoría y documentación.

✏️ 3. Permisos de modificación sobre tablas

Permitir INSERT, UPDATE, DELETE
GRANT SELECT, INSERT, UPDATE, DELETE ON Jardineria.Clientes TO 'jemarin_usu'@'192.168.10.20'; FLUSH PRIVILEGES;

Con estos permisos el usuario puede consultar y modificar registros de Clientes.

Revocar permisos de modificación
REVOKE INSERT, UPDATE, DELETE ON Jardineria.Clientes FROM 'jemarin_usu'@'192.168.10.20'; FLUSH PRIVILEGES;

Tras esta operación, el usuario conserva el permiso SELECT pero ya no puede alterar datos.

🔬 4. Privilegios a nivel de columna

Uso de columns_priv

Permite conceder permisos sobre columnas específicas. Por ejemplo, permitir modificar solo el teléfono:

GRANT UPDATE (Telefono) ON Jardineria.Clientes TO 'jemarin_usu'@'192.168.10.20';

Para ver el registro generado en las tablas del sistema:

SELECT * FROM mysql.columns_priv WHERE User = 'jemarin_usu'\G

Esta granularidad es clave para aplicar el principio de mínimo privilegio.

🚀 5. Cheatsheet de Comandos

Comandos Esenciales de MySQL
  • SHOW DATABASES; - Lista todas las bases de datos.
  • USE [database]; - Selecciona una base de datos para trabajar.
  • SHOW TABLES; - Muestra las tablas de la base de datos actual.
  • DESCRIBE [table]; - Muestra la estructura de una tabla.
  • SELECT * FROM [table] LIMIT 10; - Muestra las 10 primeras filas de una tabla.
  • SHOW GRANTS FOR 'user'@'host'; - Muestra los privilegios de un usuario.
  • SELECT user, host FROM mysql.user; - Lista todos los usuarios y sus hosts.
  • FLUSH PRIVILEGES; - Recarga los privilegios desde las tablas de sistema.

🧠 6. Buenas prácticas en gestión de permisos

Resumen de Seguridad
  • Mínimo privilegio: conceder únicamente los permisos necesarios.
  • Restringir hosts: especificar la IP o rango de IPs del usuario.
  • Usar vistas: para limitar el acceso a datos sensibles (columnas o filas).
  • Revocar permisos: eliminar privilegios que ya no se necesiten.
  • Auditar regularmente: consultar las tablas `mysql.user`, `mysql.db`, `tables_priv`, etc.
  • Contraseñas seguras: usar contraseñas largas y complejas.
