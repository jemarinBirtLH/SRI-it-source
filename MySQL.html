1. Conexion y entorno base

Primeros comandos al entrar
mysql -u root -p SHOW DATABASES; SELECT VERSION(); STATUS; SHOW VARIABLES LIKE 'character_set_server';

Antes de tocar permisos, comprueba version, codificacion y base activa para evitar errores de contexto.

2. Creacion de usuarios

Usuario administrador local
CREATE USER 'jemarin_adm'@'localhost' IDENTIFIED BY 'Adm_2026_Fuerte!'; GRANT ALL PRIVILEGES ON *.* TO 'jemarin_adm'@'localhost' WITH GRANT OPTION; FLUSH PRIVILEGES;

Este usuario debe quedar limitado al propio servidor para tareas de administracion.

Usuario de aplicacion (host restringido)
CREATE USER 'jemarin_app'@'192.168.10.20' IDENTIFIED BY 'App_2026_Robusta!'; GRANT SELECT ON Jardineria.* TO 'jemarin_app'@'192.168.10.20'; FLUSH PRIVILEGES; SELECT user, host FROM mysql.user WHERE user = 'jemarin_app';
Tip: evita '%' salvo casos controlados y auditados.

3. Vistas sobre la base de datos

Crear vista para ocultar tablas crudas
USE Jardineria; CREATE OR REPLACE VIEW vista_pedidos_clientes AS SELECT p.CodigoPedido, p.FechaPedido, p.Estado, c.NombreCliente, c.Telefono FROM Pedidos p JOIN Clientes c ON p.CodigoCliente = c.CodigoCliente;

La vista entrega datos listos para consumo sin exponer todas las columnas internas.

Conceder acceso solo a la vista
GRANT SELECT ON Jardineria.vista_pedidos_clientes TO 'jemarin_app'@'192.168.10.20'; FLUSH PRIVILEGES; SHOW GRANTS FOR 'jemarin_app'@'192.168.10.20';
Esperado: SELECT * FROM vista_pedidos_clientes; funciona y SELECT * FROM Clientes; devuelve error de permisos.

4. Permisos de modificacion

Conceder y revocar por tabla
GRANT SELECT, INSERT, UPDATE, DELETE ON Jardineria.Clientes TO 'jemarin_app'@'192.168.10.20'; FLUSH PRIVILEGES; REVOKE INSERT, UPDATE, DELETE ON Jardineria.Clientes FROM 'jemarin_app'@'192.168.10.20'; FLUSH PRIVILEGES;

Aplica permisos minimos y revisalos periodicamente con SHOW GRANTS.

5. Privilegios por columna

Limitar UPDATE a un campo concreto
GRANT UPDATE (Telefono) ON Jardineria.Clientes TO 'jemarin_app'@'192.168.10.20'; SELECT Host, Db, User, Table_name, Column_name, Column_priv FROM mysql.columns_priv WHERE User = 'jemarin_app';

Con esto el usuario puede editar solo telefono sin tocar nombre, direccion ni credito.

6. Como hacer busquedas en MySQL

Busqueda exacta

SELECT * FROM Clientes WHERE NombreCliente = 'Euro+ Shopping Channel';

Rapida y precisa cuando conoces el valor exacto.

Busqueda parcial con LIKE

SELECT NombreCliente, Telefono FROM Clientes WHERE NombreCliente LIKE '%Garden%';

Util para filtros tipo "contiene" en cuadros de busqueda.

Busqueda por rango de fechas

SELECT CodigoPedido, FechaPedido, Estado FROM Pedidos WHERE FechaPedido BETWEEN '2008-01-01' AND '2008-12-31';

Comun para informes mensuales o anuales.

Busqueda con orden y limite

SELECT NombreCliente, LimiteCredito FROM Clientes ORDER BY LimiteCredito DESC LIMIT 10;

Muestra top resultados sin cargar toda la tabla.

Busquedas avanzadas
-- Texto libre con FULLTEXT (si la tabla/columna lo permite) SELECT NombreCliente FROM Clientes WHERE MATCH(NombreCliente) AGAINST ('flores jardin' IN NATURAL LANGUAGE MODE); -- Patron por expresion regular SELECT NombreCliente FROM Clientes WHERE NombreCliente REGEXP '^[A-Z].*Shop'; -- Ver plan de ejecucion para optimizar EXPLAIN SELECT * FROM Pedidos WHERE Estado = 'Entregado';
Clave de rendimiento: crea indices para columnas de filtro frecuente, por ejemplo Estado, FechaPedido o claves de relacion.

7. Ejemplos de uso real

Casos practicos de trabajo
A) Panel comercial: pedidos por estado
SELECT Estado, COUNT(*) AS total FROM Pedidos GROUP BY Estado ORDER BY total DESC;
  • Se usa para KPI diario de ventas y logistica.
  • Permite detectar cuellos de botella en "Pendiente".
B) Alertas de clientes con credito alto
SELECT NombreCliente, LimiteCredito FROM Clientes WHERE LimiteCredito > 100000 ORDER BY LimiteCredito DESC;
  • Ayuda a priorizar seguimiento comercial.
  • Puede usarse para scoring de riesgo.
C) Auditoria de permisos de usuarios
SELECT user, host, account_locked, password_expired FROM mysql.user ORDER BY user, host; SHOW GRANTS FOR 'jemarin_app'@'192.168.10.20';
  • Permite revisar cuentas inactivas o demasiado abiertas.
  • Es parte de un checklist de seguridad mensual.

8. Cheatsheets rapidos

DDL (estructura)

CREATE DATABASE proyecto; CREATE TABLE tareas ( id INT PRIMARY KEY AUTO_INCREMENT, titulo VARCHAR(120) NOT NULL, creada_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); ALTER TABLE tareas ADD COLUMN estado VARCHAR(20); DROP TABLE tareas;

DML (datos)

INSERT INTO tareas (titulo, estado) VALUES ('Montar servidor', 'pendiente'); UPDATE tareas SET estado = 'hecha' WHERE id = 1; DELETE FROM tareas WHERE id = 1; SELECT * FROM tareas;

DCL/TCL (seguridad y transacciones)

GRANT SELECT ON proyecto.* TO 'app'@'10.0.0.15'; REVOKE INSERT ON proyecto.* FROM 'app'@'10.0.0.15'; START TRANSACTION; UPDATE cuentas SET saldo = saldo - 100 WHERE id = 1; UPDATE cuentas SET saldo = saldo + 100 WHERE id = 2; COMMIT; -- o ROLLBACK

Diagnostico rapido

SHOW PROCESSLIST; SHOW ENGINE INNODB STATUS\G SHOW INDEX FROM Clientes; EXPLAIN SELECT * FROM Pedidos WHERE Estado = 'Entregado'; ANALYZE TABLE Pedidos;

9. Glosario MySQL

Terminos clave
Schema / Base de datos
Contenedor logico que agrupa tablas, vistas, triggers y rutinas.
Tabla
Estructura principal de filas y columnas donde se guardan datos.
Vista
Consulta guardada que se comporta como tabla virtual para simplificar acceso o restringir datos.
Indice
Estructura auxiliar que acelera busquedas y joins sobre columnas seleccionadas.
JOIN
Operacion para combinar filas de dos o mas tablas usando una relacion comun.
Primary Key (PK)
Columna (o conjunto) que identifica cada fila de forma unica.
Foreign Key (FK)
Columna que referencia una PK de otra tabla para mantener integridad referencial.
Transaction
Grupo de operaciones que se confirman juntas (COMMIT) o se deshacen (ROLLBACK).
Privilegio
Permiso de seguridad como SELECT, INSERT, UPDATE, DELETE o EXECUTE.
EXPLAIN
Comando para inspeccionar el plan de ejecucion y optimizar consultas lentas.

10. Buenas practicas

Checklist recomendado
  • Minimo privilegio: otorga solo lo necesario para cada rol.
  • Host restringido: evita cuentas abiertas con '%' salvo necesidad justificada.
  • Versionado SQL: guarda scripts de schema y permisos en Git.
  • Indices medidos: crea indices en columnas de filtro frecuente y revisa con EXPLAIN.
  • Backups: define estrategia de copias y pruebas de restauracion periodicas.
  • Auditoria: revisa SHOW GRANTS y cuentas obsoletas en cada revision mensual.
