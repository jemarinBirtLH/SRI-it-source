Hacking Ético

Guía operativa para auditoría técnica profesional en redes, web, sistemas, Wi-Fi e IA.

Uso legal obligatorio: todo comando de esta guía es para laboratorios propios o auditorías con autorización escrita. No uses estas técnicas fuera de alcance pactado.
Flujo de trabajo recomendado
  1. Definir alcance y reglas del test (activos, ventanas horarias, tipo de pruebas).
  2. Reconocimiento pasivo y activo para inventario técnico.
  3. Escaneo de superficie y detección de vulnerabilidades.
  4. Validación manual de hallazgos y priorización por impacto/riesgo.
  5. Remediación, retest y cierre con evidencia.

Reconocimiento

IA en hacking ético (red teaming y defensa)

La IA acelera tanto a atacantes como a defensores. En un equipo ético se usa para mejorar detección, priorización y simulaciones, nunca para acciones fuera de alcance.

  • Generación de hipótesis de ataque y casos de prueba.
  • Análisis de grandes volúmenes de logs con clasificación automática.
  • Red teaming de aplicaciones LLM (prompt injection, data exfiltration, jailbreaks).
  • Asistencia en hardening y validación continua en CI/CD.

Ejemplos prácticos de auditoría

nmap -sV --script=vulscan/vulscan.nse 192.168.56.10 # Resultado esperado: # - servicios/versiones expuestos # - CVE asociados para plan de parcheo

Usa perfiles de escaneo autenticado y no autenticado. Compara resultados tras aplicar parches para medir reducción de riesgo.

Flujo: 1. Configurar navegador con proxy local (127.0.0.1:8080) 2. Interceptar login y parámetros de entrada 3. Repetir peticiones con cambios controlados 4. Validar autenticación, sesión, encabezados y filtrado
sudo airmon-ng start wlan0 sudo airodump-ng wlan0mon # Revisar: # - cifrado (WPA2/WPA3) # - WPS desactivado # - potencia/cobertura no excesiva # - clientes desconocidos

Complementa con Kismet y captura en Wireshark para analizar tramas de gestión y detectar comportamientos anómalos.

Objetivo: medir si el bot filtra datos sensibles o ejecuta acciones no permitidas por prompt injection.

# Ejemplo con garak (modelo local/API configurado) garak --model_type openai --model_name gpt-4o-mini # Ejemplo con promptfoo (suite de pruebas YAML) promptfoo eval promptfoo redteam run

Buenas prácticas: dataset de pruebas versionado, criterios de aceptación, y bloqueo de despliegue si falla seguridad.

Pipeline típico: SIEM - enriquecimiento de IOC - clasificación asistida por LLM - validación humana final.

Resultado esperado: - Menos falsos positivos - Triage más rápido - Reglas Sigma/EDR mejoradas con feedback continuo
Ruta de estudio recomendada: 1) Linux + redes, 2) web/API, 3) AD, 4) Wi-Fi, 5) IA security y LLM red teaming.
