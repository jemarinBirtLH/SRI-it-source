Iniciacion completa: CSS desde cero

Aqui empiezas sin suponer conocimientos previos. Si dominas este bloque, ya puedes construir interfaces limpias, conectar estilo con HTML y controlar estados visuales desde JavaScript.

1) Sintaxis minima de CSS

Una regla siempre tiene selector, propiedad y valor.

selector {
  propiedad: valor;
}

2) Clases e IDs en HTML

La clase se reutiliza; el ID se reserva para un elemento unico.

<div class="card">...</div>
<h2 id="title-main">...</h2>

3) Selectores esenciales

Primero domina los selectores base; luego avanza a combinadores y estados.

.card {}
#title-main {}
nav a {}
button:hover {}

4) Integracion con HTML

Se recomienda CSS externo para orden, cache y mantenibilidad.

<link rel="stylesheet" href="styles.css">

5) Integracion con JavaScript

No cambies estilos sueltos: alterna clases para mantener el sistema coherente.

const box = document.querySelector(".panel");
box.classList.toggle("is-active");

6) Flujo recomendado de aprendizaje

Selector -> caja -> layout -> componentes -> arquitectura -> performance.

Base  ->  Intermedio  ->  Avanzado
HTML/CSS   Layout        Sistema UI

Laboratorio cero: HTML + CSS + JS en 3 pasos

Paso A: HTML

<button id="toggleBtn">Activar</button>
<div class="panel">Caja</div>

Paso B: CSS

.panel { opacity: .5; }
.panel.is-active { opacity: 1; }

Paso C: JavaScript

toggleBtn.addEventListener("click", () => {
  panel.classList.toggle("is-active");
});

Resultado: separas estructura, estilo y comportamiento desde el primer dia.

Que es CSS, su historia y por que evoluciono tanto

CSS (Cascading Style Sheets) es el lenguaje que controla la capa visual de la web: color, layout, tipografia, animacion y adaptacion responsive. Nacio para separar contenido y presentacion.

Resumen tecnico rapido

Antes de CSS, HTML cargaba tambien decisiones visuales. Eso hacia dificil mantener sitios grandes. Con CSS, la web gano coherencia visual, rendimiento por cache y escalabilidad para equipos.

1996: CSS1 1998: CSS2 2011+: CSS3 modular Hoy: Grid, Variables, Container Queries
CSS
Cascade Layout Motion A11y

1996

CSS1: Base del estilo web

Primer standard para fuentes, colores y espaciado. Marca la separacion entre estructura y presentacion.

1998

CSS2: posicionamiento y media types

Llegan conceptos de layout mas avanzados y estilos orientados a distintos medios de salida.

2009-2017

Era modular: CSS3 por especificaciones

Animaciones, transforms, gradients, flexbox y nuevas APIs visuales evolucionan por modulos.

2017-2026

CSS moderno de produccion

Grid, custom properties, clamp(), subgrid, container queries, :has() y mejor tooling de debug.

Modulos Base Guiados (principiante -> intermedio)

Bloque de consolidacion para aprender CSS con una secuencia clara antes del nivel avanzado.

  • Empieza por selectores de clase para evitar acoplamiento excesivo.
  • Usa IDs solo en casos puntuales (ancoras o elementos unicos).
  • Evita depender de cadenas largas de selectores descendientes.
.btn {}
.btn--primary {}
#hero-title {}
nav a:hover { text-decoration: underline; }
  • Prioriza hojas externas para cache y mantenimiento.
  • Organiza por capas: base, layout, componentes, utilidades.
  • Mantiene una jerarquia limpia de carpetas en el proyecto.
<head>
  <link rel="stylesheet" href="assets/css/base.css">
  <link rel="stylesheet" href="assets/css/components.css">
</head>
  • No escribas estilos inline desde JS salvo casos puntuales de runtime.
  • Controla estados con clases para mantener consistencia visual.
  • Centraliza la logica de interaccion en funciones pequenas.
const menu = document.querySelector(".menu");
const toggle = document.querySelector("#menuBtn");

toggle.addEventListener("click", () => {
  menu.classList.toggle("is-open");
});
  • Aprende margin vs padding antes de meterte en grids complejas.
  • Activa box-sizing: border-box para predecir tamanos.
  • Construye primero un layout simple de cabecera, contenido y footer.
* { box-sizing: border-box; }
.layout {
  display: grid;
  grid-template-columns: 220px 1fr;
  gap: 1rem;
}

Modulos Avanzados Guiados (intermedio -> experto)

Abre cada modulo para escalar desde buenas practicas hasta arquitectura profesional.

  • Separa reglas por capas: reset, base, componentes y utilidades.
  • Evita !important salvo utilidades puntuales y documentadas.
  • Define convencion de naming desde el inicio del proyecto.
@layer reset, base, components, utilities;

@layer base {
  :root { --space-3: 1rem; }
  body { margin: 0; }
}

@layer components {
  .card { padding: var(--space-3); }
}
  • Usa tokens semanticos (--surface-primary, --text-muted).
  • Configura temas por atributo: [data-theme="alt"].
  • Evita codificar colores directos en componentes.
:root {
  --surface-primary: #0d1728;
  --text-primary: #e6eef9;
}

[data-theme="light"] {
  --surface-primary: #ffffff;
  --text-primary: #0f172a;
}

.panel {
  background: var(--surface-primary);
  color: var(--text-primary);
}
  • Disena por componentes, no solo por viewport.
  • Aplica container-type para adaptar tarjetas por ancho real.
  • Usa subgrid para alinear cabecera, cuerpo y footer en cards.
.catalog {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(18rem, 1fr));
  gap: 1rem;
}

.item {
  container-type: inline-size;
}

@container (min-width: 26rem) {
  .item__meta { display: flex; gap: .5rem; }
}
  • Usa escalas fluidas para titulares, no saltos bruscos por media query.
  • Define line-height por rol semantico (titular, body, metadata).
  • Conserva una escala de espaciado comun para todo el sistema.
:root {
  --step-0: clamp(0.95rem, 0.9rem + 0.25vw, 1.1rem);
  --step-4: clamp(2rem, 1.5rem + 2vw, 3.2rem);
}

h1 { font-size: var(--step-4); line-height: 1.05; }
p { font-size: var(--step-0); line-height: 1.65; }
  • Combina :focus-visible, :has() y estados semanticos.
  • Evita ocultar feedback de foco por estetica.
  • Controla transiciones solo en propiedades permitidas.
.field:has(input:focus-visible) {
  outline: 2px solid #45d0ff;
  outline-offset: 3px;
}

.btn {
  transition: transform .2s ease, box-shadow .2s ease;
}

.btn:hover { transform: translateY(-2px); }
  • Define un lenguaje de motion con tiempos consistentes.
  • Sincroniza easing segun tipo de accion (entrada, salida, feedback).
  • Respeta preferencias del usuario con prefers-reduced-motion.
@keyframes rise {
  from { opacity: 0; transform: translateY(12px); }
  to { opacity: 1; transform: translateY(0); }
}

.card {
  animation: rise .45s cubic-bezier(.16, .84, .44, 1) both;
}

@media (prefers-reduced-motion: reduce) {
  .card { animation: none; }
}
